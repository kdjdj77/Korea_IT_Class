오버라이딩(overriding)
	상속 과정에서 부모의 메소드를 자식에서 재정의 하는것
	부모타입의 객체에서 메소드가 사용되면 부모에서 정의된대로 실행되고
	자식타입의 객체에서 메소드가 사용되면 자식에서 재정의된대로 실행된다

오버로딩(overloading)
	메소드의 이름은 같지만 매개변수의 갯수가 다르거나 타입 다르게 정의하는 것

다형성(polymorphism)을 구현하는 요소이다
	하나의 모양으로 다르게 동작 하도록 사용하는 것

@~ : 어노테이션 주석
	컴퓨터가 보는 설명
	의도적으로 오류를 발생시킴으로서 실수를 줄일 수 있다

super
	부모클래스를 의미한다
	부모클래스 안에 있는 메소드를 사용할때는 super. 해서 사용한다

	super()는 생성자를 의미한다

자식생성자를 호출하면 반드시 부모의 생성자를 먼저 호출하고, 그 다음 자식 생성자가
호출이 된다
만약 상속을 받았는데, 오류가 난다면 부모의 기본생성자가 정의되어있는지 확인하면 된다

업캐스팅 다운캐스팅
	업캐스팅
		자식타입을 부모타입으로
		업캐스팅이 된다면 자식에서 새롭게 정의한 멤버는 사용 불가능
		하지만 자식에서 재정의한 메소드는 재정의한대로 실행이 된다

	다운캐스팅
		부모타입을 자식타입으로
		(업캐스팅 된 객체를 다시 자식타입으로 되돌리는 행위)
		못썼던 멤버들을 다시 활용할 수 있다

자바의 모든 클래스는 Object 라는 클래스를 상속받는다
Object : 최상위 클래스
